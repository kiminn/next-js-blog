## ERD와 정규화 과정

**ERD(Entity Relationsship Diagram)** 는 데이터베이스를 구축할 때 가장 기초적인 뼈대 역할을 하며,

릴레이션 간의 관계들을 정의한 것입니다.

서비스 구축 시 단단한 골조의 역할로, 가장 먼저 신경 써야 할 부분입니다.

## ERD의 중요성

ERD는 시스템 요구사항을 기반으로 작성되며, 이를 기반으로 데이터베이스를 구축합니다.

데이터베이스 구축 이후에도 디버깅 또는 비즈니스 재설계가 필요한 경우 설계도의 역할을 담당하기도 합니다.

##### 🔴 비정형 데이터를 충분히 표현할 수 없다는 단점이 있습니다.

## 정규화 과정

**정규화 과정** 은 릴레이션 간의 잘못된 종속 관계로 인해 데이터베이스 이상 현상이 일어나 이를 해결하거나,
저장 공간을 효율적으로 사용하기 위해 릴레이션을 여러 개로 분리하는 과정입니다.

**데이터베이스 이상 현상** 이란 회원이 한 개의 등급을 가져야 하는데 세 개의 등급을 갖거나 삭제할 때 필요한 데이터가 같이 삭제되고,
데이터를 삽입해야 한는데 하나의 필드 값이 NULL이 되면 안 되어서 삽입하기 어려운 현상을 말합니다.

## 프론트 엔드 개발자가 ERD에 대하여 알아봐야하는 이유

1. 데이터 모델 이해
   ERD를 통해 프론트엔드 개발자는 백엔드에서 사용되는 데이터 모델을 이해할 수 있습니다. <br/>
   이는 데이터 구조와 관계를 명확하게 파악할 수 있게 해 주며, 이로 인해 더 효율적으로 데이터를 요청하고 활용할 수 있습니다.

2. 인터페이스 설계
   개선데이터 모델의 이해는 사용자 인터페이스(UI) 설계에 도움이 됩니다. <br/>
   개발자는 데이터의 관계와 중요성을 이해함으로써 사용자가 필요로 하는 정보를 보다 직관적으로 제공할 수 있는 UI를 설계할 수 있습니다.

3. 요구 사항 분석과 커뮤니케이션
   ERD를 이해함으로써 프론트엔드 개발자는 프로젝트의 요구 사항을 더 잘 분석하고, <br/>
   백엔드 개발자나 데이터베이스 관리자와 효과적으로 의사소통할 수 있습니다.

4. 데이터 흐름과 로직의 이해
   ERD를 통해 데이터가 어떻게 흐르고, 어떤 로직에 따라 처리되는지 이해할 수 있습니다. <br/>
   이는 프론트엔드에서 데이터를 처리하고 표시하는 방식에 영향을 미칩니다.
